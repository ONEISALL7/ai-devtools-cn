# Release Checklist

发布新版本前使用这份清单。

## 版本信息

```text
版本号：
发布日期：
发布负责人：
主要变化：
```

## 发布前

- [ ] 所有目标 PR 已合并
- [ ] CI 通过
- [ ] README 仍准确
- [ ] CHANGELOG 已更新
- [ ] breaking changes 已明确标出
- [ ] 新模板已有适用场景和验收标准
- [ ] 文档没有敏感信息
- [ ] issue 中已标记本版本解决的问题

## 发布

- [ ] 创建 git tag
- [ ] 创建 GitHub Release
- [ ] 填写 release notes
- [ ] 感谢贡献者

## 发布后

- [ ] 检查 release 页面
- [ ] 更新项目 roadmap
- [ ] 创建下一版本 milestone
- [ ] 收集用户反馈
