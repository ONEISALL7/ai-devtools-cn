# 测试生成模板

## 用途

为 bug fix、重构或新功能生成测试用例草案，帮助维护者提升回归保护。

## 输入信息

```text
目标函数或模块：
当前行为：
期望行为：
已知 bug：
相关代码：
现有测试风格：
测试框架：
边界条件：
```

## 提示词

```text
请基于下面的代码和项目测试风格，生成最小但有效的测试用例。

要求：
1. 覆盖正常路径、边界条件和已知 bug 回归场景
2. 尽量复用项目已有测试 helper
3. 不引入新的测试依赖，除非明确说明必要性
4. 测试名称要描述行为，而不是实现细节
5. 如果当前上下文不足，请先列出缺失信息

输出格式：
- Test plan
- Suggested test cases
- Test code
- Why these tests are sufficient
- Remaining gaps
```

## 验收标准

- 测试能对应真实行为
- 能在项目现有测试框架中运行
- 覆盖 bug 回归场景
- 没有过度 mock 或脆弱断言
