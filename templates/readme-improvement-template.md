# README 改进模板

这个模板用于检查 README 是否能让新用户在 3 分钟内理解项目价值并开始使用。

## 复制使用

````text
你是一个开源项目 README reviewer。请根据我提供的 README 内容，指出它是否清楚说明项目价值、目标用户、快速开始、示例、贡献方式和限制。

## 项目背景

- 目标用户：
- 主要使用场景：
- 当前项目阶段：

## README 内容

```markdown
在这里粘贴 README。
```

## 需要你输出

请按以下结构输出：

1. 一句话定位：判断 README 是否能在第一屏说明项目是什么。
2. 缺失信息：列出影响用户上手的缺口。
3. 结构建议：给出推荐章节顺序。
4. 可直接替换的文案：写出可以复制进 README 的改进段落。
5. 风险提示：指出是否存在夸大、过度承诺或不清楚的限制。
6. 最小修改方案：给出不超过 5 条最优先修改项。

## 评价标准

- 新用户能否快速判断项目是否适合自己。
- 是否有复制即可运行的示例。
- 是否说明项目不适合什么场景。
- 是否清楚说明贡献入口。
- 是否避免过度营销化表达。
````

## 维护者检查点

- README 是否先讲用户问题，而不是先讲内部实现。
- 快速开始是否真的能被新用户执行。
- 是否有至少一个可复制示例。
- 是否明确项目阶段和限制。
