# 文档维护模板

## 用途

辅助维护者更新 README、API 文档、教程和 FAQ，减少文档与代码不一致的问题。

## 输入信息

```text
需要更新的文档：
相关代码变化：
用户遇到的问题：
目标读者：
必须保留的术语：
需要删除的过期内容：
```

## 提示词

```text
请根据下面的代码变化和用户反馈，更新文档草案。

要求：
1. 面向目标读者解释，不要照搬实现细节
2. 保留现有文档风格
3. 明确标出需要维护者确认的事实
4. 如果发现文档和代码不一致，请列出冲突点
5. 给出最终可提交的 Markdown

输出格式：
- Summary
- Conflicts or facts to verify
- Updated Markdown
- Suggested follow-up docs
```

## 验收标准

- 新用户能按文档完成操作
- 文档没有承诺代码不支持的行为
- 示例命令和路径可执行或明确标注为占位符
