# AI 工具导入检查清单

这份清单用于团队在导入 AI 开发工具前做风险和流程确认。

## 基础信息

```text
团队名称：
试点负责人：
试点时间：
工具名称：
使用范围：
复盘日期：
```

## 权限和数据边界

- [ ] 明确哪些仓库可以被 AI 工具访问。
- [ ] 明确哪些文件不能粘贴到 AI 工具。
- [ ] 明确 token、密钥、客户数据、生产日志不能进入提示词。
- [ ] 明确是否允许 AI 工具读取私有 issue、PR 和 CI 日志。
- [ ] 明确成员离职或试点结束后的权限回收方式。

## 允许场景

- [ ] 文档草稿
- [ ] README 改进
- [ ] issue triage
- [ ] PR review 初稿
- [ ] 测试用例建议
- [ ] CI 报错分析
- [ ] release note 草稿

## 禁止场景

- [ ] 直接修改生产配置
- [ ] 直接执行数据库迁移
- [ ] 直接处理密钥或证书
- [ ] 直接合并未经 review 的代码
- [ ] 直接关闭安全检查
- [ ] 上传客户数据或敏感日志

## 验收指标

| 指标 | 记录方式 |
| --- | --- |
| 节省时间 | 记录每类任务的人工估算时间 |
| 采纳率 | 统计 AI 建议被采纳的比例 |
| 错误率 | 统计明显错误或不可执行建议 |
| 安全事件 | 记录是否出现敏感信息风险 |
| 维护者负担 | 记录 review 是否更轻或更重 |

## 复盘问题

```text
1. 哪些场景最有效？
2. 哪些场景输出不可靠？
3. 是否出现敏感信息风险？
4. 是否需要补充团队模板？
5. 是否值得扩大试点范围？
6. 是否需要停止某些使用方式？
```
