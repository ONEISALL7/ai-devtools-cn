# AI 开发工具评估模板

## 用途

帮助团队或开源维护者评估一个 AI 开发工具是否适合进入工程流程。

## 基础信息

```text
工具名称：
使用场景：
评估日期：
评估人：
项目技术栈：
```

## 评估维度

### 工程适配

- 是否能读取项目上下文
- 是否能理解现有测试和 CI
- 是否支持代码 diff 或 PR 级 review
- 是否能遵循项目风格和约束

### 质量控制

- 是否容易产生不可验证的代码
- 是否能给出测试计划
- 是否能解释变更原因
- 是否能识别兼容性和安全风险

### 维护成本

- 是否引入新的账号、权限或依赖
- 是否有清晰的审计记录
- 是否容易被团队成员复用
- 是否支持失败回滚

### 安全和隐私

- 是否会上传敏感代码或数据
- 是否支持权限控制
- 是否适合处理私有仓库
- 是否需要额外的数据合规审查

