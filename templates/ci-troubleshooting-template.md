# CI 排错模板

这个模板用于让 AI 分析 GitHub Actions、测试失败、依赖安装失败和环境差异。请只粘贴可公开日志，不要包含 token、私有账号、客户数据或内部地址。

## 复制使用

````text
你是一个 CI 排错助手。请根据我提供的仓库背景、CI 配置片段和失败日志，判断最可能的失败原因，并给出可验证的修复步骤。

## 失败日志

```log
在这里粘贴失败日志。请先删除 token、邮箱、客户名、内部域名和私有路径。
```

## 需要你输出

请按以下结构输出：

1. 失败摘要：用 3 句话以内说明失败发生在哪里。
2. 最可能原因：按概率从高到低列出 3 个原因。
3. 需要确认的信息：列出还缺哪些上下文。
4. 修复方案：给出最小改动优先的修复步骤。
5. 验证方式：说明如何在本地和 CI 中验证。
6. 风险提示：说明哪些改动可能影响生产或 release。

## 限制

- 不要编造不存在的文件。
- 不要建议跳过测试作为最终修复。
- 如果日志不足，请明确说“不足以判断”。
- 涉及依赖升级时，必须说明回滚方式。
````

## 维护者检查点

- AI 是否准确引用了失败 step。
- AI 是否区分了直接原因和根因。
- 修复建议是否可以用最小改动验证。
- 是否避免了隐藏测试、删除检查或关闭安全策略。
