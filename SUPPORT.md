# 支持入口

这个项目主要提供中文 AI 工程维护模板、案例和本地 CLI。为了让维护者更快处理问题，请根据场景选择合适入口。

## 先确认问题类型

| 场景 | 推荐入口 |
| --- | --- |
| 不知道该用哪个模板 | 先看 [如何选择模板](docs/choose-template.md) 和 [3 分钟快速上手](docs/quickstart.md) |
| 某个模板不好用 | 提交 Template usage feedback issue |
| 需要新增或改进模板 | 提交 Template request issue |
| 想贡献匿名化真实案例 | 提交 Case study contribution issue |
| CLI 命令报错或不符合预期 | 提交 CLI issue |
| 文档有错别字、链接失效或内容过期 | 提交 Documentation issue 或直接开 PR |
| 发现安全或敏感信息问题 | 按 [安全政策](SECURITY.md) 处理，不要公开粘贴敏感内容 |

## 提交前准备

为了减少来回确认，请尽量提供：

- 使用的模板、案例或 CLI 命令
- 项目类型和技术栈的公开描述
- 你希望产出的结果，例如 PR review、issue 评论、release note 或测试清单
- 已经尝试过的命令和结果
- 可以公开的错误信息或匿名化日志

## 不要公开提交的信息

请不要在 issue、PR 或讨论中粘贴：

- API key、token、密码、cookie
- 客户名称、账号、合同、订单或个人隐私
- 未公开源码、私有日志、内部域名或内部仓库路径
- 无法公开的生产事故细节

如果问题必须依赖敏感上下文，请先抽象成公开描述，再说明哪些信息被省略。

## 维护者处理方式

维护者会优先处理：

- 能复现的 CLI bug
- 会影响多个用户的模板问题
- 能补充真实维护场景的案例
- 安全边界、隐私和误用风险
- 影响新用户上手的文档问题

如果问题更适合变成模板、案例或文档改进，维护者可能会把它拆成后续 issue。
