# Maintainers

本文件记录项目维护者及职责。公开维护者信息有助于社区理解谁负责 review、release 和项目方向。

## Primary Maintainer

- GitHub: `@ONEISALL7`
- Responsibilities:
  - Review pull requests
  - Triage issues
  - Maintain releases
  - Update documentation and templates
  - Coordinate roadmap and community feedback

## Maintainer Expectations

维护者需要：

- 定期处理 issue 和 PR
- 对模板质量和安全边界负责
- 避免收录无法复现或夸大能力的内容
- 在重要变更中保留清晰的决策记录
- 维护 release notes 和 changelog

## Becoming a Maintainer

长期贡献者可以通过以下方式成为 core maintainer：

- 连续提交高质量 PR
- 帮助 triage issue
- 改进文档结构和模板质量
- 参与 release 准备和 review
- 在社区讨论中提供稳定、建设性的判断
