# 模板质量评分表

这份评分表用于评审新增或修订的 AI 工程模板，帮助维护者判断模板是否适合收录、是否需要继续打磨，以及 PR 是否可以合并。

## 使用方式

1. 先阅读模板的适用场景、输入要求、输出格式和验收标准。
2. 按下面 7 个维度分别评分，每项 0-2 分。
3. 对低分项写出具体修改建议。
4. 根据总分和安全边界判断是否接受 PR。

## 评分标准

| 维度 | 0 分 | 1 分 | 2 分 |
| --- | --- | --- | --- |
| 适用场景 | 没有说明使用场景 | 场景存在但比较泛 | 清楚说明适用项目、角色和任务 |
| 输入信息完整性 | 缺少必要上下文 | 列出部分输入，但不完整 | 明确列出背景、约束、文件、日志或目标 |
| 输出格式清晰度 | 输出要求模糊 | 有格式要求但不稳定 | 明确要求结构、字段、示例或清单 |
| 验收标准 | 没有验收标准 | 标准偏主观 | 可以被维护者或 CI 明确验证 |
| 安全边界 | 没有安全提醒 | 提到安全但不具体 | 说明敏感信息、权限、破坏性操作和人工确认边界 |
| 可复用性 | 只能用于单个案例 | 可复用但需要大量改写 | 可被不同项目按占位符快速复用 |
| 维护成本 | 容易过期或依赖私有上下文 | 需要定期更新 | 结构稳定，维护者容易检查和修订 |

## 总分判断

| 总分 | 建议 |
| --- | --- |
| 0-6 | 暂不合并。请作者补充场景、输入、输出或安全边界。 |
| 7-10 | 可以继续 review。维护者应指出必须修改的低分项。 |
| 11-14 | 可以考虑合并。若没有安全风险，可进入最终 review。 |

## PR Review 建议

维护者 review 模板类 PR 时，可以按这个顺序检查：

- 模板是否服务真实工程任务，而不是泛泛的聊天提示词。
- 模板是否要求用户提供足够上下文，避免 AI 直接猜测。
- 模板是否把输出固定成清单、表格、补丁建议或测试计划。
- 模板是否包含安全边界，特别是 token、生产环境、删除操作和权限变更。
- 模板是否有示例或能被未来示例覆盖。

## 最低合并条件

一个模板 PR 至少应该满足：

- 总分不低于 9 分。
- 安全边界维度不低于 1 分。
- 验收标准维度不低于 1 分。
- 维护者能理解模板适合什么场景，以及不适合什么场景。

如果模板涉及生产环境、凭证、依赖升级、安全修复或自动化写操作，安全边界必须达到 2 分。

## 记录方式

维护者可以在 PR review 中留下类似记录：

```text
Template score: 11/14

- 适用场景: 2
- 输入信息完整性: 2
- 输出格式清晰度: 2
- 验收标准: 1
- 安全边界: 1
- 可复用性: 2
- 维护成本: 1

合并前建议：补充一条关于敏感信息处理的说明。
```
