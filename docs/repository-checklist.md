# 开源仓库自检清单

这份清单用于检查一个 AI 开发工具文档/模板项目是否已经具备公开维护基础。

## 基础信息

- [ ] 仓库是 public
- [ ] GitHub profile 是 public
- [ ] README 说明项目用途、受众和快速开始
- [ ] LICENSE 明确
- [ ] CONTRIBUTING 说明如何贡献
- [ ] SECURITY 说明安全问题如何报告
- [ ] MAINTAINERS 说明维护者职责
- [ ] ROADMAP 说明短中期方向
- [ ] CHANGELOG 记录版本变化

## 项目内容

- [ ] 至少 5 个可复用模板
- [ ] 至少 2 个真实示例
- [ ] 模板包含适用场景、输入、输出和验收标准
- [ ] 文档没有明显过期或夸大表述
- [ ] 不包含敏感信息、token、账号或客户数据

## 维护流程

- [ ] 有 issue template
- [ ] 有 PR template
- [ ] 有 label 规范
- [ ] 有基础 CI
- [ ] 有 release 流程
- [ ] 有 review 标准

## 维护证据

- [ ] 有持续 commit 记录
- [ ] 有 issue triage 记录
- [ ] 有 PR review 或合并记录
- [ ] 有 release tag
- [ ] 有文档更新记录
- [ ] 有用户反馈或外部引用

## 申请开源维护支持前

- [ ] 能说明自己是 primary/core maintainer
- [ ] 能说明项目为什么对用户有价值
- [ ] 能提供 stars、forks、用户反馈或引用数据
- [ ] 能说明 AI credits 如何用于真实维护工作
- [ ] 申请内容和公开仓库记录一致
